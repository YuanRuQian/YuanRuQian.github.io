有状态的组件的挂载过程

根据 React Component
可知挂载过程顺序是

  • constructor()
  • static getDerivedStateFromProps()
  • render()
  • componentDidMount()

如果你需要从 API 异步请求数据，你会用哪个生命周期函数？为什么？

componentDidMount()
在组件挂载后立刻执行（向树中插入）
需要 DOM 节点的初始化行为需要放在这里
如果这里订阅了 那么在 componentWillUnmount() 里也不要忘记接触订阅

因为当数据被获取之后，你可以在这里使用 setState() 来更新组件。此时 setState() 会再次触发渲染，但是 UI 更新只会发生一次，所以用户不会看到中间的状态。
